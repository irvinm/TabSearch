Tab Hiding Permission & Privacy

IMPORTANT

  • This extension uses the tabs.hide() API to hide tabs.
  • Firefox requires explicit user permission for this feature.

On first use of the addon

  • Per Mozilla: "The first time an extension hides a tab, the browser will tell the user that the tab is being hidden, show them how they can access the hidden tab, and give them the option of disabling the extension instead."
  • The extension will briefly hide and show the last tab in your current window.
  • This action triggers Firefox to display a privacy dialog asking you to allow or deny tab hiding with this extension.
  • This is only a workaround until the bug referenced in Note #1 is fixed.

What should you do?

  • To use tab hiding with this extension, click Keep Tabs Hidden in the dialog.
  • To disable this feature, click Disable Extension.

Note #1: If you do not accept the permission, the dialog can appear each time the extension tries to hide a tab and may cause UI conflicts between the search dialog and Firefox's privacy dialog.

Once you allow it, you can select "Disable initial hide action" in extension settings to skip this startup initialization step in the future.

  • Reference: My discussion on Mozilla's Discourse site
  • Bugzilla: Issue tracking this behavior

Note #2: "Disable initial hide action" only affects the startup permission initialization step. It does not change how matching/filtering works during a search session.

Note #3: This addon only uses hidden tabs while searching for tabs. Once you cancel the search (press ESC or click away from the dialog) or select the tab you want, all tabs are shown again. Hidden tabs are temporary and are not permanently hidden.

Note #4: This addon does not currently track or detect if another extension is already hiding tabs using tabs.hide(). If you use multiple tab-hiding extensions at the same time, this may cause conflicts or unexpected behavior.
